    Describe the MVC design pattern as it applies to web servers - start from initial HTTP request and end with writing the response.

    An HTTP request is made to a controller, which tells the model to take some action and/or update its state. The model then responds to the controller, which selects the appropriate view to be shown to the user. The view checks the model to get any information it needs, and creates an HTTP response to be sent to the user.

    Describe the pros and cons of version control.

    Version control can make working on a large project with other people much easier by making sure everyone's code is up to date, combining everyone's changes into one file, and providing a central location to get everything. It can also show who made every change so you know who to blame if something went wrong, and lets you revert to an older version if needed.

    The only real con I can think of is that if not everyone on a project knows how to use version control, they probably won't use it even if you teach them how because it takes getting used to, and if half of the team is using it and the other half isn't, it usually ends up even worse than if no one was using it at all.
